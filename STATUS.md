## Дизайн-конвенции экрана котировок (ОСАГО + Здоровье) — зафиксировано с ПМ+дизайнером

Экраны выбора СК унифицированы. Решения **залочены** (не переоткрывать без причины):

- **Табличное выравнивание.** Карточки — одинаковые фиксированные колонки (НЕ subgrid: конфликт с padding карточек). ОСАГО-сетка `64px minmax(0,1fr) 130px 320px 160px`. Контейнер: ОСАГО 1040px, Здоровье 880px, по центру.
- **«Итого» — главное число** (28px navy). Цена ОСАГО — вторичная (24px серая): ранг виден по сортировке и «?». Два числа не конкурируют.
- **«?» у цены → поповер с расчётом** по коэффициентам (ОСАГО: ТБ×КТ×КБМ×КВС×КМ×КО; Здоровье: сумма×тариф×срок×категория×кол-во−КВ). Click-toggle, закрытие по клику-вне/Esc.
- **Сортировка:** Сегмент сверху, Перестраховочный пул — всегда снизу; внутри групп по цене. Самая дешёвая — бейдж «Лучшая цена» + зелёный акцент, выбрана по умолчанию.
- **Способ оплаты** у «Итого» (ОСАГО) выводится из переключателя плательщика: Физлицо/ИП → «Оплата картой», Юрлицо → «Оплата по счёту».

- **НЕ показывать КВ агента в рублях** на экране (у проекта другая система мотивации). Подвал-«полоса доверия» убран.

## Известные проблемы / технический долг

- **Перезагрузка страницы (F5)** теперь восстанавливает agent profile из localStorage. Если переименовать поля в `AuthenticatedAgent` — старые ключи в localStorage могут оказаться без новых полей у уже залогиненных тестировщиков (нужен relogin).
- **api-contract.yaml** описывает `login` как «ИКП агента или email» — устарело, нужно обновить на «номер телефона» при следующем правке контракта.
- **Стили компонентов разбросаны по `.scss` страниц.** Пора вынести повторяющиеся блоки в `shared/components/` (`app-page-header`, `app-status-badge`, `app-data-table`; экран котировок ОСАГО/Здоровье — кандидат №1 на `app-quote-row`). `osago.page.scss` раздулся до ~19.6 kB (богатый экран: форма+загрузка+котировки+модалка+поповер+оплата+успех), из-за чего `anyComponentStyle` error-бюджет в `angular.json` поднят до **24 kB** (warning — 8 kB как напоминание). Историю: 12 → 20 → 24 kB.
- **Два разных паттерна поиска.** Таблица клиентов — серверный поиск через `toObservable` + `debounceTime` + `switchMap` (правильный). Пролонгация — клиентская фильтрация через `searchTick`-сигнал, который computed `myFiltered` читает, чтобы пересчитаться при вводе в `FormControl`. При выносе в `shared/` свести к одному подходу.
- **Оформление создаёт договор только в памяти.** `POST /policies` (`createPolicy`) кладёт полис в начало массива `policies` (in-memory, на сессию). После оплаты сразу открывается `/clients/:id` — работает в рамках одного прохода. **При F5 на странице договора полис «теряется»** (моки регенерируются при перезагрузке → «договор не найден»). Если демо с перезагрузками — нужен persist (localStorage). Имитация эквайринга — `setTimeout` 3 c; счётчики лоадера (`CALC_TOTAL_MS = 40_000` в ОСАГО, `HL_STEP_MS` в Здоровье) демонстрационные.
- **Список «Мои клиенты» не знает про новые продукты Здоровья.** Свежий договор появляется сверху, но колонка продукта/объекта рендерится по старой логике (тип `NS`/`TICK`, без авто). Причесать при доработке списка.
- **`DESIGN_SYSTEM.md`**: палитра синхронизирована с `tokens.scss` (брендовый navy+голубой, `--header-bg`, 3 роли цвета). Компонентные паттерны (шапка с аватаром, чат) добить при `/system` styleguide. Источник истины по токенам — `src/styles/tokens.scss`.
- **PWA service-worker** включён только в production-build. В dev — отключён через `isDevMode()`.

## Vite dev-server: критичный нюанс окружения

**Проект ДОЛЖЕН лежать в пути без пробелов и кириллицы** (`D:\aa-app\`, не `D:\Архив Claude\Прототип 3.0\`). Иначе Vite генерирует `polyfills.js` с unencoded путями (`/@fs/D:/Архив Claude/...`), браузер не может загрузить zone.js, рендерится белая страница. Junctions и `NODE_PRESERVE_SYMLINKS=1` не помогают (Vite внутри использует `fs.realpath`).

**`ng serve` без `--host` биндится только на IPv6 (`::1`)** на этой машине, а Windows резолвит `localhost` в IPv4 (`127.0.0.1`) первым → «сайт не открывается». Запускать **`ng serve --host 0.0.0.0 --port 4200`** (слушает IPv4) или открывать `http://127.0.0.1:4200/` напрямую.

**esbuild EPIPE после долгой сессии / новых зависимостей.** Если добавить новую библиотеку (напр. начали использовать Taiga) в долго работающий `ng serve`, esbuild может упасть (`The service is no longer running: write EPIPE`) и отдавать битый чанк — страница «не открывается». Лечится **перезапуском dev-сервера** (чистая переоптимизация зависимостей). Признак в логе сервера — строка с `EPIPE`.
