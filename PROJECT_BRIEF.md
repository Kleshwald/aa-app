# Agent Academy — Project Brief

Кликабельный прототип нового фронтенда платформы Agent Academy.

## О компании

**Agent Academy** — B2B-платформа для страховых агентов в России.
Через платформу агенты:
- Оформляют полисы нескольких типов: ОСАГО, Кросс-продукты НС при ДТП, Юрист поможет, Миникаско и отдельно НС, Антиклещ, Ипотечное страхование
- Работают с 7 страховыми компаниями
- Получают комиссию за оформление полисов
- Управляют клиентами и пролонгациями
- Проходят обучение по продуктам

**Текущая система:** монолит на 1С:Предприятие 8.3, доступ через веб-клиент по адресу b2b.agentacademy.ru.

**Сейчас:** 1500 пользователей, 300 активных в день.
**Цель:** рост до 10 000 пользователей, 2000 активных в день.

## Зачем новый фронтенд

Текущая система имеет 4 главные боли:

### 1. Поддержка продаж
Виджет чата нормально не встраивается в 1С — нет прямой коммуникации агент-компания.
Текущая Carrot Quest ломается (теряются сообщения, проблемы с уведомлениями).

### 2. Плохой UX
- Неконкурентный интерфейс
- На разных мониторах появляются лишние скроллы
- Формы подрезаются
- Особенно сложно новым агентам — обратная связь: "непонятно"
- Нет адаптивности для планшетов и телефонов
- Невозможно за несколько кликов с мобильного оформить пролонгацию

### 3. Платные лицензии 1С
Растёт количество пользователей — пропорционально растут затраты на лицензии (сейчас 50 лицензий ~250 тыс/мес).
Деньги хочется направить на маркетинг и мотивацию агентов, а не на лицензии 1С.

### 4. Нестабильность
Веб-приложение на 1С периодически подвисает — особенно тяжёлые операции (выгрузка аналитики за длинный период) при плохом интернете.

## Цель прототипа

**Это не MVP, не финальный продукт.** Это **кликабельный прототип** для:

1. **Показа собственникам** — для согласования бюджета на полноценную разработку
2. **Показа команде разработки** (1С и Angular) — для оценки трудозатрат
3. **Тестирования на 5-10 агентах** — для получения обратной связи до больших инвестиций
4. **Фиксации архитектурных решений в коде** — чтобы команда не спорила о принципах
5. **Использования как "скелет"** — реальная разработка будет строиться поверх прототипа

## Стратегическое решение

**Гибридная архитектура** (Strangler Fig):
- 1С остаётся **бэкендом** (бизнес-логика, интеграции СК, комиссии, выплаты, регуляторика)
- **Новый фронтенд на Angular** заменяет веб-клиент 1С
- Связь между фронтом и 1С через **HTTP-сервисы 1С** (REST API)
- Постепенный переход на новый стек, без big-bang переписывания

**В V1 НЕ строится:**
- Middle-слой (Node.js) — отложено, добавим в V2 при необходимости
- Замена бизнес-логики 1С
- Новые интеграции

## Целевая аудитория

См. файл **USER_PROFILE.md** для детального анализа.

**Краткий портрет:**
- Женщина 48 лет
- Живёт в небольшом городе или районе Сибири/Дальнего Востока
- Работает страховым агентом

- НЕ знакома с современными "стартап-стилями" интерфейса

## Что должно работать в прототипе

5 ключевых пользовательских сценариев:

### 1. Логин
Простой экран входа: логин + пароль + кнопка "Войти".
Без регистрации (агентов создаёт куратор).

### 2. Главная страница (дашборд)
После входа агент видит:
- Сколько заработал в этом месяце
- Сколько активных полисов
- Сколько пролонгаций
- Срочные действия (просроченные обращения, пролонгации сегодня)
- Последние оформленные полисы
- Быстрые действия (рассчитать ОСАГО, найти клиента, связаться с поддержкой)

### 3. Расчёт ОСАГО
Самый частый сценарий:
1. Агент вводит данные машины (VIN, марка, модель, год)
2. Агент вводит данные владельца (ФИО, дата рождения, дата прав)
3. Нажимает "Рассчитать"
4. Получает предложения от 7 СК с ценами и комиссией
5. Выбирает одно предложение и нажимает "Оформить"

### 4. Мои полисы
Таблица всех полисов агента:
- Фильтры: по типу, по СК, по статусу, по дате
- Поиск: по ФИО клиента, номеру полиса, номеру машины
- Сортировка по колонкам
- Действия: посмотреть детали, продлить, расторгнуть

### 5. Профиль агента
- Личные данные
- Контактные данные
- Статистика: всего полисов, заработано всего, по месяцам
- Настройки уведомлений
- Реквизиты для выплат (просмотр, без редактирования в прототипе)

## Что НЕ в скоупе прототипа

- Регистрация новых агентов
- Реальная авторизация
- Реальные платежи
- Реальные интеграции со страховыми
- Бэкенд (всё моки через interceptor)
- Чат с поддержкой (отдельный проект)
- Аналитика и дашборды для админов
- Управление пользователями (для кураторов и админов)
- Обучение
- Реальные ИНН/паспортные данные (только тестовые)

## Технические требования

См. **ARCHITECTURE.md** и **.cursorrules** для деталей.

**Кратко:**
- Angular 18 standalone components
- TypeScript strict mode
- Taiga UI 5 как UI-библиотека
- Tailwind CSS для layout
- PWA support (installable, basic offline)
- Mobile-responsive (от 320px)
- Десктоп-first (минимум 1366×768)
- Производительность: загрузка <3 сек на 3G

## Дизайн-требования

См. **DESIGN_SYSTEM.md** для деталей.

**Кратко:**
- Стиль: **профессиональный российский B2B** (как Госуслуги, Сбербанк Онлайн)
- НЕ как Linear/Vercel/Apple
- Крупный текст (18px body)
- Высокий контраст (WCAG AA минимум)
- Кнопки 48px+ для удобства попадания
- Иконки с подписями
- Минимум анимаций
- Понятные и привычные паттерны

## Команда

**Сейчас:**
- Продакт-менеджер: 1 (управление, постановка задач)
- 1С: 1 тимлид + 1 джуниор (на аутсорсе через студию)
- Angular: 2 сеньора (через агентство)
- Дизайн: Claude + Cursor (продакт работает с AI как с дизайнером)

**На этом этапе (прототип):**
- Продакт работает с Cursor самостоятельно
- 2-4 недели работы

## Сроки

**Этап 1: Подготовка** (1-2 недели) — этот документ и сопутствующие
**Этап 2: Прототип в Cursor** (3-5 недель) — сам прототип
**Этап 3: Демо и обратная связь** (1-2 недели) — показ агентам, команде
**Этап 4: Доработки** (1-2 недели) — по результатам обратной связи
**Этап 5: Передача в полноценную разработку** — после согласования бюджета

**Итого до старта реальной разработки:** ~3 месяца.

## Бюджет

См. отдельную оценку (готовится после прототипа).

**Постоянные затраты во время разработки:** ~14-20 млн ₽/год (~1.5 млн/мес)
**В это входит:**
- 1С-студия (300 тыс/мес)
- 1С мидл в штате (350-450 тыс/мес)
- 2 Angular-сеньора через агентство (800-1200 тыс/мес)
- Архитектурный надзор (80-150 тыс/мес)
- Инфраструктура (15-30 тыс/мес)

## Стратегический контекст

**Что НЕ делаем в этом проекте:**
- Не пытаемся выпилить 1С — она остаётся
- Не строим middle-слой — отложено
- Не делаем нативные мобильные приложения — только PWA
- Не делаем красивый дизайн "уровня Apple" — делаем функциональный B2B

**Что делаем:**
- Заменяем устаревший UX 1С на современный (но привычный) фронт
- Сохраняем всю бизнес-логику в 1С
- Подготавливаем архитектуру так, чтобы можно было дорастить middle-слой
- Сокращаем зависимость от лицензий 1С (фронт = 1 техн. пользователь)
- Получаем возможность мобильной работы (PWA)

## Главное

**Эта задача — про продукт, не про технологии.**

Цель — чтобы агентам было удобно работать, чтобы новые агенты быстрее адаптировались, чтобы можно было масштабироваться до 10 000 пользователей.

Технологии (Angular, Taiga, Tailwind, моки) — это **инструменты** для достижения этой цели, не самоцель.
