# Agent Academy — Профиль пользователей

Документ основан на анализе **реальных данных** платформы Agent Academy
(выгрузка по полу, возрасту, географии 914 активных агентов на 26 мая 2026).

Это **критический документ** для всех дизайн-решений.
Любая фича, любой экран, любой компонент должен **проверяться** через этот профиль.

## Кратко: основной портрет

> **Женщина 48 лет**, страховой агент в небольшом городе или районе
> Красноярского края/Иркутской области/Алтайского края, работает
> в своём офисе или на дому. Использует старый ноутбук с Windows
> и Chrome/Yandex. Хорошо знакома с Госуслугами и Сбербанк Онлайн.
> К современным "стартап-стилям" интерфейса (Linear, Notion) не привыкла.

## Технологический профиль (выводы из географии и возраста)

Из географии + возраста + статистики продаж страховых услуг можно с высокой вероятностью предположить:

### Устройства

- **Десктоп — основной канал** (по типу работы — оформление полисов с множеством полей)
- Ноутбуки/ПК **в среднем старше 5-7 лет** (для регионов это типично)
- Типичные характеристики:
  - CPU: Intel Core i3-i5 (3-7 поколений)
  - RAM: 4-8 GB
  - Экран: 1366×768 - 1920×1080 (преобладает первое)
  - ОС: Windows 7, Windows 10 (реже Windows 11)
- Мобильные устройства: 
  - Android — преобладает
  - Айфоны — в меньшинстве
  - Версии не последние

### Браузеры

- Chrome — преобладает (общая статистика)
- Yandex Browser — высокий процент в РФ
- Edge — некоторая доля
- Safari — только на айфонах (меньшинство)
- Firefox — редкость

### Интернет

- В районах часто **нестабильный** (3G/4G через сотового оператора)
- **Не везде** есть оптоволокно
- В деревнях/малых городах — иногда долгая загрузка страниц

## Цифровая грамотность

### С чем привыкли работать

- **Госуслуги** (новая версия) — почти все
- **Сбербанк Онлайн** (личный кабинет для физлиц)
- **Word/Excel** (для отчётов)
- **WhatsApp/Telegram/MAX** (для общения)
- **Старая 1С Agent Academy** (текущая система)

### С чем НЕ привыкли работать

- Современные SaaS-инструменты (Notion, Linear, Slack)
- Adobe-продукты (Figma, и т.п.)
- Codebase-инструменты (GitHub)
- "Современные" UX-паттерны (свайпы, скрытые меню, hamburger по-разному)

### Уровень компьютерной грамотности

- **Средний и ниже среднего**
- Умеют: оформлять полис, заполнять формы, работать с почтой
- НЕ умеют: настраивать сложные параметры, разбираться в технических ошибках, использовать "интуитивные" современные UX

## Психологический профиль

Это **рабочие пользователи**, не "потребители":
- Система — это **инструмент** для зарабатывания денег
- Не нужны "вау-эффекты"
- Нужна **скорость, надёжность, понятность**
- **Не любят сюрпризов** — каждая смена интерфейса = стресс
- **Боятся ошибок** — особенно когда дело про деньги (комиссии, выплаты)

**Главные мотивации:**
1. Заработать больше денег (оформить больше полисов)
2. Быстро решать проблемы клиентов
3. Не тратить время на технические сложности

**Главные страхи:**
1. Случайно сделать что-то не то (отменить полис, ввести неверные данные)
2. Потерять данные клиента
3. Не понять, что произошло (когда что-то "сломалось")
4. Подвести клиента

## Что это значит для дизайна

### Размеры (увеличенные)

| Элемент | Размер |
|---------|--------|
| Основной body-текст | **18px** (не 16) |
| Кнопки высота | **48px** минимум |
| Поля ввода | **48px** высота |
| Touch-targets (мобиль) | **48×48px** минимум |
| Иконки в кнопках | 24px (не 16) |
| Высота строки таблицы | **56px** (комфортно) |

### Контрастность

- **WCAG AA минимум**, AAA желательно
- Текст: только тёмные оттенки (gray-700 / gray-900)
- Никаких gray-400 для важной информации
- Никаких "стильных" низкоконтрастных серых

### Понятность

- Кнопки с **текстовыми подписями** (не только иконки)
- Иконки **с подписями** под/рядом
- **Хлебные крошки** на всех страницах
- **Заголовки страниц** крупные и явные
- **Состояния всегда видны** (не "появляются по hover")

### Предсказуемость

- Логотип всегда **слева сверху** и ведёт на главную
- Поиск всегда **справа сверху**
- Кнопка "Назад" доступна
- Меню **видно сразу**, не скрыто за бургером (на десктопе)
- Кнопки действий **в одном порядке** (Отмена — слева, Сохранить — справа)

### Терпимость к ошибкам

- **Двойное подтверждение** для опасных действий
- **Undo** для всех изменений (30 секунд)
- **Чёткие сообщения об ошибках** на русском
- **Автосохранение** форм (черновики)

### Производительность

- **Малый размер бандла** (приоритет)
- **Минимум JavaScript** там, где можно без него
- **Кэширование** через Service Worker
- **Прогрессивная загрузка** (Skeleton states)

### Адаптивность

- **Десктоп-first** (основной канал)
- **Минимум 1366×768** (старые ноутбуки)
- **Мобильная версия** — для отдельных сценариев (пролонгации в выходной)

## Что НЕ делать (антипринципы)

1. ❌ **Не делать как Linear/Vercel/Apple** — слишком "технологично" для аудитории
2. ❌ **Не использовать сленг и англицизмы** — "submit", "loading", "OK" — это плохо
3. ❌ **Не делать тёмную тему по умолчанию** — пожилым сложнее
4. ❌ **Не использовать тонкие шрифты** (300 weight) — нечитаемо
5. ❌ **Не делать "креативные" интерфейсы** — нужна предсказуемость
6. ❌ **Не использовать иконки без подписей** — непонятно
7. ❌ **Не скрывать важные функции в меню** — должно быть видно сразу
8. ❌ **Не делать быстрых анимаций** (>200ms — раздражают)
9. ❌ **Не использовать жесты как основной способ** (свайпы и т.п.)
10. ❌ **Не делать "оптимистичные" интерфейсы** без подтверждения

1. **Госуслуги** (gosuslugi.ru) — новая версия 2023-2024 годов

   - Простой B2B-интерфейс

## Антиреференсы (как НЕ надо)

1. ❌ **Linear (linear.app)** — слишком "для разработчиков"
2. ❌ **Notion** — слишком гибкий, слишком много опций
3. ❌ **Vercel Dashboard** — слишком "технологичный"
4. ❌ **Apple Music/Spotify** — эмоциональные, не функциональные
5. ❌ **Современные стартап-лендинги** (parallax, big text) — пугают
6. ❌ **Текущая 1С Agent Academy** — устаревший UX (как НЕ должно быть)

## Финальный чек-лист для дизайна

Перед тем как утверждать любой компонент или экран, проверь:

- [ ] Елена 48 сможет понять, что делать?
- [ ] Татьяна 56 сможет прочитать без напряжения?
- [ ] Дмитрий 34 не будет раздражён "медленностью"?
- [ ] Светлана 67 сможет нажать кнопку с первого раза?
- [ ] На разрешении 1366×768 всё видно?
- [ ] При медленном интернете загружается?
- [ ] При случайном клике можно отменить?
- [ ] Если что-то пошло не так, понятно почему?

Если хотя бы один ответ "нет" — переделываем.
