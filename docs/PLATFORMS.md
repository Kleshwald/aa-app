# Версии и платформы

Документ-намерение по будущим вариантам кабинета агента. Сейчас всё — **в разработке (планирование)**, кода нет. Источник истины для разделов `/hub/doc/mobile`, `/hub/doc/tablet`, `/hub/doc/react`.

Прототип на **Angular** остаётся эталоном UX. Любая другая версия повторяет его поведение и проходит через те же законы: [USER_PROFILE.md](../USER_PROFILE.md) (аудитория 45+, старый ноутбук, 3G) и [DESIGN.md](../DESIGN.md) (подписи сверху, ≥16px текст, ≥44px контролы, ошибки текстом+иконкой).

## Что переносимо между платформами

- **Дизайн-токены** — `src/styles/tokens.scss` (цвета, отступы, радиусы, типографика).
- **Контракт API** — `api-contract.yaml` (одни и те же эндпоинты и модели).
- **Доменные факты** — раздел в `/hub/doc/domain` и `CLAUDE.md`.

Меняется только слой UI-компонентов.

## Мобильная версия

- **Зачем:** агенты работают «в полях», со смартфона быстрее, чем с ноутбука. Ключевые сценарии: расчёт ОСАГО, статус полиса, оплата, сообщения.
- **Подход (черновик):**
  1. Адаптив текущего Angular-прототипа (один код, мобильные раскладки).
  2. PWA «установить на экран» — `@angular/pwa`/manifest уже подключены.
  3. Нативная обёртка (Capacitor) — только если понадобятся push / камера / скан документов.
- **Статус:** планирование. Приоритет и сроки не зафиксированы.

## Версия для планшета

- **Зачем:** планшет — частый «второй экран» на встрече с клиентом; места больше, чем у телефона.
- **Подход (черновик):** та же адаптивная сетка, отдельный breakpoint между мобильным и десктопом; формы в две колонки; крупные контролы и шрифты сохраняем (DESIGN.md не смягчаем).
- **Статус:** планирование, после мобильной адаптации.

## Вариант на React

- **Зачем:** команда разработки заказчика может быть сильнее в React. Прототип на Angular при этом остаётся эталоном UX, React-вариант повторяет его 1:1.
- **Подход (черновик):** переиспользуем токены и контракт API; пилот на 1–2 экранах для оценки трудозатрат; решение по полному переносу — по итогам пилота.
- **Статус:** оценка. Решение не принято.
