# Agent Academy — Дизайн-система

## Философия

**Для кого:** страховые агенты 35-60 лет, преимущественно женщины (68%),
работающие в регионах России (97%) на устаревших компьютерах с Windows.

См. **USER_PROFILE.md** для подробного анализа аудитории.

## 6 главных принципов

### 1. Знакомость > Современность
Делаем как в Госуслугах, Сбербанк Онлайн, Тинькофф ИП — **не** как в Linear/Vercel/Notion.
Привычные паттерны = быстрая адаптация.

### 2. Чёткость > Эстетика
**Высокий контраст**, крупный шрифт, явные границы элементов.
Никаких "стильных" низкоконтрастных серых.

### 3. Предсказуемость > Инновация
Каждый элемент работает одинаково на всех страницах.
Никаких сюрпризов, скрытых функций, "интуитивных" жестов.

### 4. Информативность > Минимализм
Кнопки **с текстом** (не только иконки).
Иконки **с подписями**.
Состояния явные.

### 5. Производительность > Анимации
Работаем на старых компьютерах и медленном интернете.
Минимум анимаций (≤150ms), без тяжёлой графики.

### 6. Десктоп-first > Mobile-first
Основной канал — десктоп с минимум 1366×768.
Мобильная версия — для отдельных сценариев (пролонгации, статусы).

|-------|----------|---------------|
| 0 | 0px | — |
| 0.5 | 2px | Hairline-отступы |
| 1 | 4px | Минимальные |
| 2 | 8px | Между связанными элементами |
| 3 | 12px | Внутри кнопок, мелкий padding |
| **4** | **16px** | **Стандартный padding** |
| 5 | 20px | Между группами |
| **6** | **24px** | **Padding карточек (desktop)** |
| 8 | 32px | Между секциями |
| 10 | 40px | Большие отступы |
| 12 | 48px | Между крупными блоками |
| 16 | 64px | Hero-секции |
| 24 | 96px | Очень большие (редко) |

### Типичные комбинации

- Padding кнопок: `12px 24px` (base), `16px 32px` (large)
- Padding inputs: `12px 16px`
- Padding карточек: `16px` (mobile), `24px` (desktop)
- Padding модалок: `32px`
- Gap между карточками в сетке: `16px` (mobile), `24px` (desktop)

## АНИМАЦИИ

**Минимум.**

### Длительность

| Класс | Значение | Использование |
|-------|----------|---------------|
| `fast` | 100ms | Hover, focus (мгновенно) |
| `base` | 150ms | Стандартные переходы |
| `slow` | 200ms | Модалки (максимум) |

**Никогда не более 250ms.** Раздражает.

### Easing

- `ease-out` для входов (modal opens, dropdown opens)
- `ease-in` для выходов
- `ease-in-out` для большинства случаев

### Что анимируем

- ✅ `opacity` (быстро, производительно)
- ✅ `transform: translate/scale` (производительно)
- ✅ `color`, `background-color` (для hover)

### Что НЕ анимируем

- ❌ `width`, `height` — прыгает layout
- ❌ `margin`, `padding` — прыгает layout
- ❌ `top`, `left` — медленно
- ❌ Анимации при загрузке страниц
- ❌ Bounce, elastic, spring effects — "детский" вид

### Поддержка `prefers-reduced-motion`

Все анимации должны **отключаться**, если пользователь установил:
```css
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; }
}
```

## КОМПОНЕНТЫ

### Кнопки

| Размер | Высота | Padding (H) | Текст |
|--------|--------|-------------|-------|
| sm | 40px | 16px | 16px |
| **base** | **48px** | **24px** | **16px** |
| lg | 56px | 32px | 18px |

**Минимум для touch:** 44px (Apple guidelines). Мы используем 48px для запаса.

#### Варианты

- **Primary:** `brand-500` фон, белый текст, **bold 500**
- **Secondary:** `gray-700` фон, белый текст
- **Tertiary (outline):** белый фон, `brand-500` border 2px, `brand-500` текст
- **Ghost:** transparent, `gray-700` текст
- **Danger:** `error-600` фон, белый текст

#### Состояния

- **Hover:** темнее на 1 уровень (brand-500 → brand-600)
- **Active:** темнее на 2 уровня
- **Focus:** видимый ring (3px brand-100)
- **Disabled:** `opacity: 0.5`, `cursor: not-allowed`
- **Loading:** spinner вместо текста, disabled

#### Правила

- ✅ **Всегда с текстом** (никаких чистых иконок в кнопках)
- ✅ Если есть иконка — слева от текста
- ✅ Кнопка действия — большая и заметная
- ✅ Текст конкретный: "Оформить полис", не "Готово"
- ❌ Кнопки тоньше 40px
- ❌ Текст кнопок меньше 16px

### Поля ввода

```
Высота: 48px (base)
Padding: 0 16px
Текст внутри: 16px (НЕ 14px)
Border: 2px (заметнее чем 1px)
Border-color: gray-300 (контрастнее чем gray-200)
Focus: border brand-600, ring 4px brand-100
Border-radius: 6px
```

#### Лейблы и подсказки

- **Лейбл:** ВСЕГДА над полем (не floating, не placeholder)
  - Шрифт: 14px, `gray-700`, вес 500
- **Подсказка:** под полем, 14px, `gray-600`
- **Ошибка:** под полем, 14px, `error-600`, иконка слева
- **Placeholder:** только как пример формата (не замена лейбла)

#### Правила

- ❌ **НЕ использовать floating labels** (label-перепрыгивающий) — путает
- ❌ **НЕ использовать placeholder вместо label** — забывается при заполнении
- ❌ Поля тоньше 44px
- ✅ Маски для известных форматов (телефон, дата, VIN)
- ✅ Автофокус на первое поле формы

### Карточки

```
Фон: белый
Border: 1px solid gray-200 (явная граница)
Border-radius: 8px (mobile), 12px (desktop)
Padding: 16px (mobile), 24px (desktop)
Shadow: shadow-xs (очень тонкая) или без тени
Hover: shadow-sm (если кликабельна) + border gray-300
```

### Таблицы

**Критично:** основной способ работы с данными.

```
Шрифт в ячейках: 16px (НЕ 14)
Высота строки: 56px (комфортно для глаз и попадания)
Полоски (zebra): обязательно (gray-50 / white)
Hover: gray-100 (хорошо заметный)
Заголовки колонок: bold, 14px, gray-700, uppercase
Padding ячейки: 12px вертикально, 16px горизонтально
Sticky header: при скролле — обязательно
```

#### Требования

- ✅ Сортировка по клику на колонку (с явной стрелкой)
- ✅ Фильтры над таблицей (видимые, не в выпадайке)
- ✅ Pagination или infinite scroll (выбрать одно)
- ✅ Empty state с понятным сообщением
- ✅ Loading skeleton при загрузке

### Модальные окна

```
Минимальная ширина: 480px (sm), 640px (md), 800px (lg)
Padding: 32px
Заголовок: 24px, bold
Кнопки внизу: главная справа (Отмена слева, Сохранить справа)
Backdrop: rgba(0,0,0,0.5)
Border-radius: 16px
```

#### Правила

- ✅ Кнопка "Отмена" — слева, в gray-стиле
- ✅ Главная кнопка — справа, в brand-стиле
- ✅ Крестик в правом верхнем углу — дополнительно
- ✅ Закрытие по Escape работает
- ✅ Клик по backdrop закрывает (с подтверждением, если есть несохранённые изменения)
- ❌ Модалки внутри модалок
- ❌ Модалки без заголовка

### Уведомления (Toasts)

```
Позиция: правый верхний угол (desktop), снизу (mobile)
Длительность: 5 секунд (auto-dismiss)
Возможность закрыть вручную: всегда
Несколько одновременно: до 3 (потом стек)
```

#### Варианты

- **Success:** зелёный border-left, белый фон, текст
- **Warning:** оранжевый border-left
- **Error:** красный border-left + дольше показывать (8 сек)
- **Info:** синий border-left

### Skeleton loaders

При загрузке списков, карточек, страниц — показывать skeleton (плейсхолдер серой формы) вместо спиннера.

Спиннер только для **коротких операций** (отправка формы).

### Empty states

Когда нет данных — **информативное** пустое состояние:
- Иконка (большая, 64px, gray-400)
- Заголовок: "У вас пока нет полисов"
- Описание: что это значит и что делать
- Кнопка действия: "Оформить первый полис"

## ПРАКТИЧЕСКИЙ ЧЕК-ЛИСТ

Перед утверждением любого экрана/компонента проверь:

### Читаемость
- [ ] Основной текст 18px или больше?
- [ ] Контраст текста ≥7:1?
- [ ] Нет ли важной информации в gray-400 и светлее?

### Удобство
- [ ] Кнопки минимум 48px высотой?
- [ ] Touch-targets 48×48px минимум?
- [ ] Иконки имеют подписи?
- [ ] Можно работать только клавиатурой?

### Понятность
- [ ] Заголовок страницы явно виден?
- [ ] Хлебные крошки есть?
- [ ] Главное действие понятно с первого взгляда?
- [ ] Текст на русском, без техжаргона?

### Адаптивность
- [ ] Работает на 1366×768?
- [ ] Работает на мобильном (320-639px)?
- [ ] Текст не подрезается?
- [ ] Кнопки не выходят за края?

### Производительность
- [ ] Страница загружается на 3G за <3 сек?
- [ ] Анимации не дольше 200ms?
- [ ] Размер JS-бандла адекватный?

### Состояния
- [ ] Loading state?
- [ ] Empty state?
- [ ] Error state?
- [ ] Success feedback?

### Согласованность
- [ ] Кнопки выглядят как на других экранах?
- [ ] Цвета из палитры?
- [ ] Отступы из spacing-системы?
- [ ] Шрифты из типографики?

Если хотя бы один пункт "нет" — **дорабатываем**.
